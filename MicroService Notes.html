MicroServices Notes



Monolithic Application:


The Monolithic application represents Single Codebase - Where the application will have different layers like Presentation Layer, Service Layer , Data layer etc as a single war/jar file for deployment.
Approach is Centralized Governance, the modules are tightly coupled as one single war/jar file.

Other Drawbacks:
- As features are added, war/jar file becomes larger and difficult to maintain.
- For a small change whole Application needs to be redeployed.
- One issue can bring down the entire application/service to halt.

MicroServices:


Micro Services architecture is a variant of Service Oriented Architecture where Application is broken into Independently Developed, Independently Deployable and Independently Testable services.
Based on the Business Capabilities services are organized. These services talk to each other over the network to fulfill a request.
Since the approach is Decentralizing control/governance, Changes to one service don't always require changing all other services. When change is required in a certain part of the application, only the related service can be modified and redeployed. No need to modify and redeploy the entire application.

Other Benefits:
- Easy for Maintaining and Deploying.
- Since this is a modular approach, Business re-usability is achieved.
- Different technologies can be adopted for different MicroServices.

Drawbacks:
- Complexity increases as the Micro Services increases.
- Influences Network Latency because of frequent interaction between services.
- Debugging becomes challenging as the control flows between services.

Other Differences:
Monolithic Application MicroServices
No Latency in Monolithic as no all calls are Local. Latency increases as the the number of Micro Services increases.
In Monolithic applications the logging will happen in a single file - Easy to check and understand. Logging happens in different microservices, Takes time to trace a execution flow.
Scaling is easy , since a copy of an application has to be deployed. Complexity increases as it is not easy system to administer (Example in Kubernetes Cluster).
Turn around time for Monolithic Application takes time since the entire application has to be deployed. More Reliable since bring up the replacement service will be quick.
Throughput is faster in Monolithic as services doesn't have network latency. Compared to Monolithic , micro services might deliver lesser throughput.
