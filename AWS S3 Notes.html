AWS S3 Notes



* Amazon Simple Storage Service - to Store and Retrieve any amount of data at any time, from anywhere on the web.

* S3 stores data as objects. An Object consists of a file and optionally any metadata(Data about data). Buckets are containers for objects. When we upload a file, we can set the permissions on the object as well as any metadata.

* For each bucket, we can control access to Create / to Delete / to List / to Update, view access logs for it or choose region.

* Once we create a bucket, we cannot change the name.

aws s3 command format:

aws s3 <command> [arguments]

command - can be ls(list), cp(copy), rm(remove), sync, mv(move), mb(make bucket), rb(remove bucket), website and etc.
arguments - Separated by space. S3 URI format:

s3://<bucketname>/[<prefix/folder>]/<object>


Examples:

1. Listing all S3 buckets or Listing the contents of specific S3 buckets :

aws s3 ls

aws s3 ls s3://example_bucket_name


2. Creating a bucket : (Must be Unique across S3)

aws s3 mb s3://example_unique_bucket


3. Removing a bucket : (By defaut, To perform this task bucket should be empty)

aws s3 rb s3://bucket_to_be_deleted

aws s3 rb s3://bucket_to_be_deleted --force


4. High-Level commands : (ls, cp, rm and mv - Works just like Unix commands. Sharing only Copy and Remove examples)

aws s3 cp localFile.txt s3://destination_bucket_name

aws s3 cp s3://source_bucket_name/s3File.txt .

aws s3 cp s3://source_bucket_name/ . --recursive

aws s3 rm s3://old_bucket_name/old_file.txt

aws s3 rm s3://old_bucket_name --recursive


* Exclude and Include Filters can be used for processing objects in pattern format:

aws s3 cp s3://source_bucket_name/s3File.txt . --exclude "*" --include "*.txt" (Copies only txt extension files)

aws s3 rm s3://old_bucket_name --recursive --exclude "*" --include "*.csv" (Deletes only csv extension files)


* Move S3 bucket to different location / Region :

aws s3 sync s3://old_bucket_name s3://new_bucket_name --source-region us-west-1 --region us-west-3


5. Managing Objects : (Gives permission to view / write on an object in S3 bucket)

Syntax:

--grant <permission>=<grantee_type>=<grantee_id>

Here,
permission - Can be read, readacl, writeacl or full.
grantee_type - cri, emailId or id.

Example:

--grant read=uri=http://acs.amazonaws.com/groups/global/AllUsers
