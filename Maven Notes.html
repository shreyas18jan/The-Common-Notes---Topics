Maven Notes




Maven is the Project Management + Comprehension tool that provides a complete build lifecycle framework.

Maven provides developers ways to manage the following:
 - Builds
 - Documentation
 - Reporting
 - Dependencies
 - SCMs
 - Releases
 - Distribution
 - Mailing List

To summarize, Maven simplifies and standardizes the project build process.

Maven uses CONVENTION OVER CONFIGURATION, developers are not required to create a build process themselves because of the default project structure.
 - Source Code : ${basedir}/src/main/java
 - Resources : ${basedir}/src/main/resources
 - Test : ${basedir}/src/test
 - Compiled Byte Code : ${basedir}/target
 - Distributable JAR : ${basedir}/target/classes

POM(Project Object Model) is the fundamental unit of work in maven. It is an XML file that resides in the base directory of the project as pom.xml.

POM contains information about the project and various configuration detail used by maven to build the project.

All POM files require the project element/tag and three mandatory tags : groupId, artifactId and version.
 - project tag : We need to specify the basic schema settings such as apache schema and w3.org specifications etc.
 - Model version : Model version should be 4.0.0
 - groupId : Id of project's group. THis is generally unique in the organization or in project.
 - artifactId : Id of project. This is generally name of the project.
 - version : Version of the project.

Along with the groupId, the artifactId defines the artifact's location within the repository.

Along with the groupId, the version is used within an artifact's repository to separate versions from each other.

Command used to load the default configurations of the super POM -mvn help:effective-pom

Maven will make use of default project source folders structure, output directory, plugins required, repositories, reporting directory while executing the desired goal.

Build lifecycle is a well defined sequence of phases, which defines the order in which the goals are ti be executed. Hence phase represents a stage in maven life cycle.

Maven has 3 standard life cycles:
 - Clean
 - default (or build)
 - site

Typical maven build life cycle consists of following sequences of phases:
 - Prepare resources : Resource copying
 - Validate : Validating the information.
 - Compile : Compilation.
 - Test : Testing.
 - Package : Packaging.
 - Install : Installation
 - deploy : deploying

Maven site plugin is generally used to create fresh documentation - reports, deploy site, etc.

The repository is a directory where all the project jars, library jar, plugins, or any other project-specific artifacts are store and can be used by maven easily.

Local repository is a folder location on your machine. It gets created when you run any maven command for the first time.

Central repository is a repository provided by the maven community. It contains a large number of commonly used libraries.
when maven does not find any dependency in the local repository, it starts searching in the central repository - http://repo1.maven.org/maven2/

Maven provides the concept of the remote repository, which is the developer's own custom repository containing required libraries or other project jars.

If dependency is not available in any of the remote / Central repository, maven allows external dependencies :
<dependencies>
 <dependency>
  <groupId>ldap-jdk</groupId>
  <artifactId>ldap-jdk</artifactId>
  <scope>system</scope>
  <version>1.0</version>
  <systemPath>${basedir}/src/lib/ldapjdk.jar</systemPath>
 </dependency>
</dependencies>

Build profile is a set of configuration values, which can be used to set or override the default values of maven build.
Using a build profile, you can customize build for different environments such as production/development environment, etc.

Maven build profile can be activated in various ways:
 - Explicitly using command console input.
 - Through maven settings.
 - Based on environmental variables.
 - OS Settings.
 - Based on existence of a file/s.

Environment specific files looks like this:
 - env.properties
 - env.test.properties
 - env.prod.properties

To define property in maven:
<properties>
 <wls.minor.version>10.1.5</wls.minor.version>
 <wls.major.version>12.2.1</wls.major.version>
</properties>
To activate profile from console command : mvn test -ptest
under %USER_HOME%/.m2/settings.xml, need to add this
<activeProfiles>
 <activeProfile>test</activeProfiles>
</activeProfiles>
A plugin generally provides set of goals : mvn [plugin-name]:[goal-name]

Maven uses archetype plugins to create projects:
 mvn archetype:generate
  -DgroupId=com.example.com
  -DartifactId=exampleCase
  -DarchetypeArtifactId=maven-archetype-quickstart
  -DinteractiveMode=false
