Cucumber is a tool that supports BDD. Here, the executable specifications are written in plain text called Gherkin (set of grammar rules). Gherkin files are stored with .feature extension.

Specifications consists of multiple scenarios, each scenario is list of steps. Each step talks about the action to be performed on it.

Step definition helps Gherkins steps to connect to programming code. Step definition carries out the action mentioned in the step.


@When("^User Login$") public void user_login_pressed() throws Throwable {  driver.findElement(By.xpath(".//*[@id='login']")).click(); }


Gherkin uses keywords to give the structure and meaning to the executable specifications. Following are the keywords:
1. Feature
2. Example (or Scenario)
3. Given, When, Then, And, But for steps (or *)
4. Background
5. Scenario Outline (or Scenario Template)
6. Examples (or Scenarios)
7. Rule

There are a few secondary keywords as well:
1. """ (Doc Strings)
2. | (Data Tables)
3. @ (Tags)
4. # (Comments)

Either spaces or tabs may be used for indentation. The recommended indentation level is two spaces.


Feature: Is prime number?  # This example checks whether the number is prime or not  Scenario: Check whether 27 is prime number or not   Given start the Prime Checker   When input value is 27   Then entered number is a prime number  Scenario: Check whether 25 is prime number or not   Given start the Prime Checker   When input value is 25   Then entered number is not a prime number


Here, after keyword each statement is matched to a code block called Step Definition.

1. Feature:
- To group related scenarios.
- Provides a high-level description of a software feature.
- Should always be the First keyword in the document. Description is optional.


Feature: Is prime number?  Prime numbers are special numbers, greater than 1, that have exactly two factors, themselves and 1.  19 is a prime number. It can only be divided by 1 and 19.  The prime numbers below 20 are: 2, 3, 5, 7, 11, 13, 17, 19.


2. Example:
- The keyword Scenario is a synonym of the keyword Example.
- Collection of steps(actions to be carried out). Recommended to have not more than 5 steps.
- Pattern reference : Describe an initial context (Given steps), Describe an event (When steps) and Describe an expected outcome (Then steps).

3. Steps:
- Each step starts with Given, When, Then, And, or But.
- In Sequence, one at a time, steps are executed in a scenario. During execution, the step definition is searched.
- Keywords are not taken into account when looking for a step definition.

 > Given : Puts the system in a known state before the user (or external system) starts interacting with the system (in further steps).
 > When : To describe an event, or an action. This step is interacting with the system, or it can be an event triggered by another system.
 > Then : To describe an expected outcome, or result. This is our Assertion step.
 > And, But : We use these keywords If you have successive GivenÃ¢â‚¬â„¢s, WhenÃ¢â‚¬â„¢s, or ThenÃ¢â‚¬â„¢s.
 > * : Helpful when you have some steps that are effectively a list of things.

4. Background:
- A Background allows you to add some context to the scenarios that follow it.
- It can contain one or more Given steps, which are run before each scenario.
- A Background is placed before the first Scenario/Example, at the same level of indentation.

5. Scenario outline and Examples:
- The Scenario Outline keyword can be used to run the same Scenario multiple times, with different combinations of values.
- A Scenario Outline must contain an Examples keyword.
- The steps can use <> delimited parameters that reference headers in the examples table. Cucumber will replace these parameters with values from the table before it tries to match the step against a step definition.


 Scenario Outline: Is prime number?   Given start the Prime Checker   When input value is <input>   Then entered number is a <result> number  Examples:   | input | result |   | 27 | prime |   | 25 | not prime|


Doc Strings are handy for passing a larger piece of text to a step definition.

Data Tables are handy for passing a list of values to a step definition.

Language header on the first line of a feature file tells Cucumber what spoken language to use - for example # language: fr for French. If you omit this header, Cucumber will default to English (en).

CucumberOptions for execution - Using @CucumberOptions annotations in Runner Class, we can achieve few more customization.

1. Plugin option : different formatting options for the output reports can be changed here.
- plugin = {"pretty"} : For additional colors and stack traces for errors.
- plugin ={"html:target/html/"} : Generates a HTML report at the mentioned location.
- plugin = {"json:target/cucumber.json"} : File contains all the information recorded in json format during the execution. This can be used later for any reporting enhancements.
- plugin = {"junit:target/cucumberReport.xml"} : File contains all the information recorded in xml format during the execution. This can be used later for any reporting enhancements.
plugin option allows any / all above entries can be used at the same time.

2. dryRun option : Checks whether corresponding Step definition exists or not for a step written in feature file. By default value is false.

3. strict option : If there is no stepDefinition present, should the testing be proceeded or not. By default value is true.

4. monochrome option : Makes Console output of Cucumber execution more readable. By default value is false.

5. features option : Cucumber checks the feature files in the mentioned directory.

6. glue option : Similar to features option, Cucumber checks the StepDefinition files in the mentioned directory.

7. Snippets option : Generates code snippets in Underscore style(SnippetType.UNDERSCORE) or CamelCase style(SnippetType.CAMELCASE). Default is Underscore style.

8. tags option : Can be used when specifying what tests to run.
- tags ={"@Regression"} : Runs only tests which has Regression.
- tags ={"@Regression","@Functional"} : Runs list of tests that needs to be executed.
